# API 测试执行器

这是一个技能，用于在 Claude Code 中执行 Apifox 的 API 自动化测试。

## 工作流程

1.  **识别测试场景**：从用户的自然语言指令中，识别出要执行的“测试场景”或“测试套件”名称。
2.  **构建CLI命令**：根据识别的信息，自动构建 Apifox CLI 命令。
3.  **执行测试**：运行构建好的命令。
4.  **分析结果**：捕获 CLI 的输出，并以清晰、友好的方式向用户总结测试结果（成功/失败数量、失败原因等）。

## 执行示例

- **用户输入**：“用开发环境跑一下登录接口的测试”
- **你的行动**：你应该自动执行 `apifox run "登录接口测试" --env dev` 命令。
- **用户输入**：“跑一下支付模块的全部测试”
- **你的行动**：你应该自动执行 `apifox run -d "支付模块"` 命令。

## 注意事项

- 确保命令中的“测试场景”或“测试套件”名称是准确的。
- 如果用户没有指定环境，可以默认为 `test` 环境，并提醒用户。
- 测试完成后，重点解析日志中的错误信息，给出直观的分析。
